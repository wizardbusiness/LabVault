Related Terms: 
-  **Problem-Agitate-Solve (PAS)** structure
- **Solution-first**

**Questions**
a. Should a social media post about a solution to a problem lead with the problem or the solution?
b. Should it be split into multiple posts for problem and solution, or should it be a single post?

---

A. Lead with the problem. See Problem-Agitate-Solve (PAS) structure
	- Establishes shared context
	- signals "I've thought about this"
	- Invites agreement (or disagreement), which creates engagement. 
	- Primes users for need for resolution - they will be thinking about the problem and are more likely to engage with future topical posts. 
	- Qualifies the audience 

```
Post option A: "Here's the documentation system I built..."
Post option B: "Modern documentation is broken. Here's what I mean — and what I did about it."
```

**Decomposition**

- `The frustration post` — establishes shared context. It signals "I've thought about this deeply enough to have opinions." It invites agreement, which creates engagement and primes the reader to _want_ the solution. Crucially, it also **qualifies the audience** — people who don't care about documentation self-select out before you even introduce the system.
- `The system reveal` — the payoff. Its perceived value is almost entirely dependent on how well the problem has been set up. A solution without a felt problem is just a feature list.
- `The sequencing` — introduces a narrative arc across posts. This is how you build _followers_ rather than one-time readers. Each post creates appetite for the next.

**Recomposition** `Frustration post` + `earned credibility` + `primed audience` — leading with the problem transforms your system reveal from "here's a thing I made" into "here's the answer to something you already felt was broken," which is the difference between a product announcement and a solution people actually want to hear about.
